## Raison d'être du module
La raison d'être du module `static` est de contenir les fichiers qui sont utilisés directement par la logique de l'application, et ne sont pas modifiés par le client. Ces fichiers sont compilés une fois, et ne sont pas mis à jour pendant que l'utilisateur utilise l'application.

## Les principaux fichiers
Le fichier principal est `style.css`. Ce fichier a la responsabilité de définir les règles pour effectuer la mise en page visuelle de l'application. Par exemple, c'est dans le fichier que la couleur de la police est définie.

## Dépendaces ou hypothèses
Ce module n'a pas de dépendances.
